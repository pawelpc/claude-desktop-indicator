Claude Desktop Indicator

A tiny always-on-top window for Windows that shows which mode (Chat / Cowork / Code) and which model Claude Desktop is using right now — readable from the corner of your eye.

View on GitHub Install
Try it — this is exactly what the real window shows.

Peripheral-vision fast

Cool background colors for modes, warm shapes for model families — the two channels never collide, so you don't need to read anything.

Read-only by design

Detection uses Windows UI Automation — the same public accessibility API screen readers use. No injection, no network sniffing, no internals.

Stays out of the way

Frameless, hidden from the taskbar and Alt-Tab, draggable anywhere, remembers its spot — on any monitor.

One double-click

The shortcut starts Claude Desktop if needed (Store or direct install) and the indicator follows it — closing itself when the app closes.

The visual encoding

Three data dimensions, three visual channels:

Background → modeColor
Chat#1E40AF deep blue
Cowork#7C3AED violet
Code#0E7490 teal
Center shape → model familyColor
Opus — circle#DC2626 red
Sonnet — diamond#EA580C orange
Haiku — triangle#16A34A green
Fable — pentagon#CA8A04 gold

The version number ("4.8", "5", …) is drawn inside the shape, and a plain-text summary sits underneath. Anything that can't be read shows as a gray "?" instead — the indicator never crashes because a UI element moved.

Install

Requirements: Windows 10 (1903+) or 11 · Claude Desktop · Python 3.10+ if running from source.

git clone https://github.com/pawelpc/claude-desktop-indicator
cd claude-desktop-indicator
pip install -r requirements.txt
python src/main.py

Or build a standalone exe and point a desktop shortcut at it:

pip install pyinstaller
pyinstaller --onefile --noconsole --name ClaudeIndicator --icon assets/icon.ico src/main.py

Full instructions — including the shortcut snippet and flags like --no-launch — are in the README.

How it works

A background thread polls the Claude Desktop window's accessibility tree every 500 ms through Windows UI Automation and reads three public signals: the sidebar's Home/Code pills (via aria-current), the composer's Chat/Cowork toggle or the page URL (depending on whether the session has started), and the composer's model selector button. State changes redraw the 240×240 window in under a second.

Nothing is written, clicked, injected, intercepted, or transmitted — the tool observes exactly what a screen reader could, and stores nothing but its own window position in a local config.json.
